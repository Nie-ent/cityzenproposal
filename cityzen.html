01 — Executive Summary

CityZen คืออะไร

CityZen คือ City Intelligence Platform ที่สร้างบนโครงสร้างของ Thunder Core Platform เป้าหมายคือเป็นช่องทางกลางให้ประชาชนเข้าถึงข้อมูลเมืองแบบ Real-time — ตั้งแต่สถานะน้ำมัน ภัยพิบัติ น้ำท่วม ไปจนถึงข้อมูลสาธารณูปโภคต่าง ๆ ในพื้นที่ที่ตนเองอยู่

สำหรับ MVP รอบนี้ โฟกัสอยู่ที่ 2 ส่วนหลัก: (1) Admin Dashboard สำหรับเจ้าหน้าที่ จัดการข้อมูลปั๊มน้ำมัน และ (2) Public Map Interface สำหรับประชาชนทั่วไปดูสถานะแบบ Real-time

02 — MVP Scope

ขอบเขต MVP

MVP แบ่งออกเป็น 2 Product Track ที่ทำงานบนฐานข้อมูลและ API ชุดเดียวกันของ Thunder Core

Track A — Admin Dashboard (Staff Portal)

หน้าจัดการสำหรับ Tenant Staff (เจ้าหน้าที่ปั๊มหรือผู้ดูแลระบบ) ในการอัปเดตและบริหารข้อมูลปั๊มน้ำมัน

Station Management — จัดการข้อมูลปั๊มน้ำมัน Admin
  • ตั้งค่า Location ของปั๊ม (lat/lng) พร้อม Map Picker ผ่าน Mapbox
  • กำหนดข้อมูลพื้นฐาน: ชื่อ, แบรนด์, จังหวัด, เขต, ที่อยู่, ช่วงเวลาเปิด-ปิด
  • เปิด/ปิดสถานะปั๊ม (is_active) และบริหาร Station profile
  • กำหนดรายการน้ำมันที่ให้บริการ (Fuel Types: 91, 95, E20, Diesel ฯลฯ)
API Endpoints ที่ใช้
GET /api/fuel/stations — ดึงรายการปั๊มทั้งหมดของ Tenant
POST/PUT /api/fuel/stations — สร้าง/แก้ไขข้อมูลปั๊ม
GET /api/fuel/types — ดึงประเภทน้ำมันทั้งหมด
Fuel Status Management — อัปเดตสถานะน้ำมัน Admin Realtime
  • อัปเดตสถานะแต่ละประเภทน้ำมัน: มี / หมด / ราคา (PATCH per fuel type)
  • Dashboard แสดง Last Updated timestamp ของแต่ละ fuel type
  • Quick Toggle UI — อัปเดตได้เร็วด้วย toggle + price input per row
  • ประวัติการอัปเดตสถานะย้อนหลัง (timestamp log)
API Endpoints ที่ใช้
PATCH /api/fuel/stations/:id/status — อัปเดต isAvailable + price
GET /api/fuel/stations/:id — ดึงสถานะปัจจุบัน
Traffic & Queue Management — จำนวนผู้ใช้บริการ Admin
  • อัปเดตระดับความหนาแน่น: ว่าง / ปานกลาง / แน่น (low / medium / high)
  • แสดงสถิติ Traffic รายชั่วโมงจาก traffic_reports
  • ดูรายงานจากประชาชน (user_reports) ที่ส่งเข้ามาเกี่ยวกับปั๊มนั้น
API Endpoints ที่ใช้
GET /api/fuel/stations/:id/traffic — ข้อมูลจราจรรอบปั๊ม
GET user_reports (Supabase Realtime subscribe)

Track B — Public Interface (Citizen App)

หน้าสำหรับประชาชนทั่วไปไม่ต้อง Login สามารถเปิดดูข้อมูลในพื้นที่ที่ตนอยู่ได้ทันที

Interactive Map — แผนที่ปั๊มน้ำมัน (Mapbox) Public Realtime
  • แผนที่ Mapbox แสดง Marker ของปั๊มน้ำมันรอบตำแหน่งที่ผู้ใช้อยู่
  • Marker สี: เขียว = มีน้ำมัน, แดง = หมด, เทา = ไม่ทราบสถานะ
  • Cluster marker เมื่อ zoom ออก — แสดงจำนวนปั๊มในพื้นที่
  • คลิก Marker เปิด Popup แสดงสถานะน้ำมันแต่ละประเภท + ราคา
  • Filter: แบรนด์ / ประเภทน้ำมัน / "มีน้ำมัน" เท่านั้น
  • Real-time update เมื่อสถานะน้ำมันเปลี่ยน (Supabase Realtime)
API Endpoints ที่ใช้
GET /api/fuel/stations?province=X&available=true — ดึงปั๊มในพื้นที่
GET /api/fuel/types — ประเภทน้ำมัน
SUB Realtime: table=station_fuel_status (INSERT/UPDATE)
Situation Cards — การ์ดสถานการณ์ Public Realtime
  • แสดง Event Cards แนวนอน (Horizontal Scroll) ด้านล่างแผนที่
  • ประเภทการ์ด MVP: ⛽ น้ำมันในพื้นที่ | 🌊 น้ำท่วม | ⚠️ ภัยพิบัติ | 🚧 ถนนปิด
  • สรุปสถิติ: ปั๊มในรัศมี 5km / กี่แห่งมีน้ำมัน / ปั๊มไหนใกล้ที่สุด
  • การ์ดคลิกได้ — เปิด Detail sheet หรือ Focus บนแผนที่
  • ข้อมูลสถานการณ์จาก user_reports และ station_fuel_status แบบ Real-time
Citizen Reporting — รายงานจากประชาชน Public
  • ไม่ต้อง Login — รายงานสถานะน้ำมันได้ทันทีจาก Popup ปั๊ม
  • รายงาน: น้ำมันหมด / มีน้ำมัน / คิวยาว พร้อมโน้ตเพิ่มเติม
  • Crowdsource ช่วยเติมเต็มข้อมูลในช่วงที่ Staff ยังไม่ได้อัปเดต
API Endpoints ที่ใช้
POST /api/fuel/reports — ส่งรายงาน (Public, ไม่ต้อง Auth)
          { stationId, fuelTypeId, reportType, note }
reportType: "out_of_stock" | "available" | "long_queue"

Webhook Server — Implementation

Webhook ถูก implement เป็น Next.js API Route ที่ /api/line/webhook รับ POST จาก Line Platform

Webhook Handler Logic Server-side
  • Verify signature ด้วย LINE_CHANNEL_SECRET ทุก request (HMAC-SHA256)
  • Parse event type: postback (จาก Rich Menu) | message | location
  • Route ตาม event.postback.data หรือ text keyword
  • เรียก Thunder Core API ด้วย Service Token (ไม่ใช่ User JWT)
  • Build Flex Message / Carousel / Quick Reply แล้ว reply ด้วย replyToken
  • Respond 200 OK ทันที (Line timeout = 30s) ถ้า slow query ใช้ async + push message แทน
Environment Variables เพิ่มเติม
LINE_CHANNEL_ACCESS_TOKEN=xxxx # สำหรับ reply/push message
LINE_CHANNEL_SECRET=xxxx # สำหรับ verify webhook signature
LINE_LIFF_ID=xxxx # LIFF App ID สำหรับหน้า Map + Report Form
SUPABASE_SERVICE_ROLE_KEY=xxxx # Server-side API calls (ไม่ expose)
LIFF (Line Front-end Framework) Integration Public
  • Public Map Interface จะ register เป็น LIFF App — เปิดใน Line ได้เลยแบบ in-app browser
  • liff.init() ดึง Line User Profile (userId, displayName) โดยไม่ต้อง Login แยก
  • Citizen Report Form ใช้ LIFF เพื่อผูก userId กับ report (ป้องกัน spam)
  • หน้าแผนที่ LIFF ส่ง location กลับผ่าน liff.sendMessages() ได้
  • LIFF URL format: https://liff.line.me/{liffId} ใส่ใน Rich Menu action ได้ทันที

08 — Boundaries

ขอบเขตที่ไม่รวมใน MVP

สิ่งเหล่านี้อยู่ใน Roadmap แต่ไม่ถูกนำมาพัฒนาใน MVP รอบนี้

  • Push Notification / Line Notify เมื่อน้ำมันหมดในพื้นที่ (v2)
  • Disaster / Flood Layer — ข้อมูลภัยพิบัติจากหน่วยงานภายนอก (v2)
  • Navigation / Routing ไปยังปั๊ม (อาจใช้ Deep link ไป Google Maps แทน)
  • User Account สำหรับประชาชน (Favorite stations, Report history)
  • Analytics Dashboard สำหรับ Admin (Station usage, Report trends)
  • Mobile Native App (iOS/Android) — MVP จะเป็น PWA ก่อน
  • Multi-language (EN/TH) — MVP เป็น TH เท่านั้น
